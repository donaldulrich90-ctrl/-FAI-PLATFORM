Faso ISP Manager

Plateforme de gestion multi-sites pour opérateurs d'accès Internet — ventes Wi-Fi, monitoring réseau, facturation et interventions terrain.

1 Présentation de la plateforme

Faso ISP Manager est une application web conçue pour les fournisseurs d'accès Internet (FAI) opérant des hotspots Wi-Fi et des liaisons point-à-point. Elle centralise en un seul endroit la gestion des ventes de tickets Wi-Fi, le suivi des revenus, la surveillance du réseau et la coordination des interventions terrain.

📡

Monitoring réseau

Carte interactive des sites, statut des liaisons PtP, intégration Zabbix et Uptime Kuma.

🎫

Ventes Wi-Fi Zone

Génération de vouchers (codes QR), gestion des revendeurs, commissions automatiques, export PDF.

💰

Finance & Rapports

Journal de caisse, rapports journaliers par revendeur, exports Excel et CSV pour la comptabilité.

🔧

Interventions terrain

Tickets de panne mobiles (mobile-first), assignation aux techniciens, suivi des statuts.

🏢

Multi-organisations

Architecture SaaS : plusieurs FAI indépendants sur la même instance, données isolées.

⚙️

Automatisation

Cron intégré (django-q2) : rapports auto, sync MikroTik, mise à jour santé PtP toutes les 10 min.

2 Rôles et accès

Chaque utilisateur appartient à une organisation (tenant) et possède l'un des trois rôles suivants :

Administrateur

  • Accès complet à tous les modules
  • Gestion des utilisateurs et revendeurs
  • Vue de tous les rapports et exports
  • Configuration des sites et équipements
  • Export caisse (CSV)
  • Interface Django Admin

Technicien

  • Tableau de bord monitoring
  • Tickets d'intervention terrain
  • Mise à jour du statut des pannes
  • Accès aux liaisons PtP sur la carte

Revendeur

  • Dashboard personnel (ventes & commissions)
  • Consultation de ses rapports journaliers
  • Export Excel de ses propres rapports
  • Génération de tickets (via l'admin)
ℹ️

Un opérateur plateforme (superutilisateur) peut voir les données de toutes les organisations et gérer les tenants depuis /plateforme/organisations/.

3 Connexion à la plateforme

Accédez à l'application via votre navigateur à l'adresse fournie par votre administrateur (ex. http://votre-serveur:8000).

  1. Page de connexion Saisissez votre nom d'utilisateur et votre mot de passe dans le formulaire. Cliquez sur Se connecter.
  2. Redirection automatique Selon votre rôle, vous êtes redirigé vers la page d'accueil adaptée :
    — Admin / Technicien → Tableau de bord monitoring
    — Revendeur → Espace Revendeur
  3. Navigation La barre latérale gauche affiche les menus accessibles selon votre rôle. En bas de la sidebar, votre nom d'utilisateur et le bouton de déconnexion sont visibles.
⚠️

Si vous voyez une page 403 — Accès refusé, votre rôle ne vous autorise pas à accéder à cette section. Contactez votre administrateur.

4 Tableau de bord réseau

Accès : / — Réservé aux rôles Administrateur et Technicien.

Le tableau de bord donne une vue temps réel de l'infrastructure réseau.

4.1 Carte interactive

La carte Leaflet affiche automatiquement tous vos sites et liaisons point-à-point. La couleur indique l'état de santé :

CouleurÉtatSignification
VertUP — OKLiaison opérationnelle, RSSI ≥ −65 dBm
OrangeDÉGRADÉLiaison faible, RSSI entre −65 et −75 dBm
RougeDOWNLiaison hors service ou RSSI < −75 dBm
GrisINCONNUDonnées Zabbix non disponibles

Cliquez sur un marqueur de site pour voir son identifiant. Les lignes entre sites représentent les liaisons PtP.

4.2 Uptime Kuma

Le bloc Uptime Kuma affiche le statut JSON de votre page de supervision (services web, DNS, etc.). Les données sont rafraîchies à chaque chargement de page.

4.3 Accès rapide

Des raccourcis permettent d'accéder directement aux Interventions terrain et à l'API Zabbix depuis le tableau de bord.

💡

La santé des liaisons PtP est mise à jour automatiquement toutes les 10 minutes via le planificateur django-q2. L'état visible sur la carte correspond à la dernière synchronisation Zabbix.

5 Ventes Wi-Fi Zone (Tickets vouchers)

Le module Wi-Fi Zone gère la vente de tickets d'accès Internet via hotspot MikroTik.

5.1 Comprendre les tickets

DuréeCodeUsage
3 heures3hConnexion courte (café, marché)
24 heures1dJournée complète
7 jours1wSemaine
30 jours30jAbonnement mensuel

Chaque ticket a un code unique (ex. KONE-A4F2B). Le préfixe identifie le revendeur. Un QR code est généré pour chaque ticket.

5.2 Générer un lot de tickets (Admin)

  1. Ouvrir l'Admin Django → /admin/wifi_zone/wifiticketbatch/add/
  2. Remplir le formulaire : Libellé du lot, Site concerné, Durée, Prix unitaire (XOF), Quantité (max 500).
  3. Enregistrer. Les tickets sont générés automatiquement avec des codes uniques.
  4. Assigner un revendeur. Dans la liste des tickets générés, définissez Revendeur / vendeur pour chaque ticket (ou par lot via l'action groupée).
  5. Exporter en PDF. Sélectionnez les tickets → Action Exporter les tickets sélectionnés en PDF. Imprimez et distribuez.
ℹ️

Après création, les tickets sont automatiquement provisionnés sur le routeur MikroTik du site correspondant via SSH (si la configuration hotspot est renseignée).

5.3 Export PDF des tickets

Deux méthodes d'export PDF sont disponibles :

MéthodeURLDescription
Par lot/wifi/tickets/lot/<id>/pdf/Tous les tickets d'un lot
Sélection/wifi/tickets/pdf/Tickets cochés dans l'admin

Le PDF généré contient le code du ticket, le QR code scannable, la durée et le prix. Format optimisé pour l'impression A4 (plusieurs tickets par page).

5.4 Commissions revendeurs

Les commissions sont calculées automatiquement à la création de chaque ticket :

Commission = Prix × Taux revendeur ÷ 100
Exemple : 500 XOF × 10% = 50 XOF commission
Net FAI = 500 − 50 = 450 XOF

Le taux de commission est défini par revendeur dans son profil utilisateur (champ Commission par défaut). Il peut être surchargé par ticket.

5.5 Dashboard Revendeur

Accès : /wifi/revendeur/

Chaque revendeur dispose d'un tableau de bord personnel montrant :

  • Nombre total de tickets vendus
  • Son taux de commission
  • Montant total à reverser au FAI (net)
  • Les 25 dernières ventes avec code, durée, site et statut

6 Abonnés Wi-Fi Simple

Le Wi-Fi Simple gère des abonnements mensuels avec contrôle par adresse MAC (pas de voucher, accès permanent plafonné à la date d'expiration).

6.1 Créer un abonné (Admin)

  1. Admin Django → /admin/wifi_zone/wifisimplesubscriber/add/
  2. Remplir : Nom complet, Téléphone, Adresse MAC du CPE (format AA:BB:CC:DD:EE:FF), Date d'expiration, Site, Équipement cible.
  3. Enregistrer. La MAC est automatiquement autorisée sur le bridge MikroTik ou Ubiquiti via SSH.

6.2 Bloquer / Débloquer un abonné

Lorsque le paiement n'est pas à jour, passez le champ Abonnement payé à jour à Non. La MAC est automatiquement bloquée sur le routeur. Le déblocage s'effectue en repassant à Oui.

ChampValeurEffet réseau
Abonnement payé à jourOuiMAC autorisée — accès ouvert
Abonnement payé à jourNonMAC bloquée via bridge filter SSH
⚠️

La commande SSH vers le routeur est exécutée en temps réel lors de la sauvegarde. Vérifiez que l'équipement cible est accessible depuis le serveur. Consultez le champ MAC bloquée sur le réseau pour confirmer l'état réel.

7 Tickets d'intervention terrain

Accès : /terrain/interventions/ — Rôles Technicien et Administrateur.

Interface mobile-first pour la gestion des pannes réseau sur le terrain.

7.1 Cycle de vie d'un ticket

OUVERT → ASSIGNÉ → EN COURS → RÉSOLU → FERMÉ
StatutSignificationAction suivante
OUVERTPanne signalée, non assignéeAssigner un technicien
ASSIGNÉTechnicien désignéDémarrer l'intervention
EN COURSIntervention en coursMarquer résolu
RÉSOLUProblème régléClôturer
FERMÉTicket archivé—

7.2 Créer un ticket de panne (Admin)

  1. Admin Django → /admin/finance/maintenanceticket/add/
  2. Remplir : Titre, Description, Site concerné, Liaison PtP défectueuse (optionnel), Priorité (Basse / Moyenne / Haute), Technicien assigné.
  3. Enregistrer. Une référence unique (PANNE-XXXX) est générée automatiquement.

7.3 Mettre à jour un ticket (Technicien)

  1. Depuis le terrain, ouvrez /terrain/interventions/ sur votre téléphone.
  2. Cliquez sur le ticket assigné à votre nom.
  3. Changez le statut via le menu déroulant et cliquez Enregistrer.
  4. Quand le problème est résolu, passez le statut à RÉSOLU. La date de clôture est enregistrée automatiquement.
💡

Un technicien ne voit que les tickets qui lui sont assignés ou non assignés. L'administrateur voit tous les tickets de l'organisation.

8 Rapports et exports financiers

8.1 Rapports journaliers revendeurs

Accès : /rapports/revendeurs/ — Admin voit tous les revendeurs, Revendeur voit uniquement les siens.

Un rapport est généré automatiquement chaque nuit à 01h00 pour chaque revendeur actif. Il récapitule :

  • Nombre de tickets vendus et utilisés dans la journée
  • Montant brut encaissé (XOF)
  • Commission du revendeur (XOF)
  • Net à reverser au FAI (XOF)
  • Détail ticket par ticket (code, durée, site, prix, statut hotspot)

Exporter en Excel

Depuis la liste des rapports, choisissez une plage de dates et cliquez sur le bouton Excel. Le fichier téléchargé contient :

OngletContenu
Export listeUne ligne par rapport avec totaux en bas (bleu)
Résumé (détail)Fiche financière du rapport (revendeur, dates, montants)
Tickets (détail)Détail complet de chaque ticket vendu ce jour
💡

Pour régénérer manuellement un rapport manqué : python manage.py generate_daily_revendeur_reports --date 2026-06-15

8.2 Export journal de caisse (CSV)

Accès : Lien Export Caisse (CSV) dans la barre latérale — Administrateurs uniquement.

Exporte toutes les écritures du journal de caisse (entrées et dépenses). Paramètres optionnels via l'URL :

# Export de toute la période
/rapports/caisse/export/csv/

# Export avec filtre de dates
/rapports/caisse/export/csv/?date_from=2026-06-01&date_to=2026-06-30

Le fichier CSV s'ouvre directement dans Excel (séparateur ;, encodage UTF-8 avec BOM). Colonnes : Date, Type, Catégorie, Description, Montant XOF, Site, Créé par, Date saisie.

8.3 Journal de caisse (saisie)

Accès : /admin/finance/cashjournalentry/add/

Saisissez chaque mouvement de caisse :

ChampDescription
TypeEntrée (recette) ou Dépense
Montant (XOF)Montant en Franc CFA
DescriptionLibellé de l'opération
CatégorieEx. carburant, maintenance, vente ticket…
Date comptableDate de l'opération (pas nécessairement aujourd'hui)

10 Installation locale (développement)

  1. Prérequis Python 3.12+, pip. Optionnel : Git.
  2. Cloner le projet
    git clone <repo-url> faso_isp
    cd faso_isp
  3. Créer l'environnement virtuel et installer les dépendances
    python -m venv .venv
    .venv\Scripts\activate # Windows
    pip install -r requirements.txt
  4. Configurer les variables d'environnement Copiez .env.example en .env et adaptez les valeurs. Pour le développement local, activez SQLite : USE_SQLITE=1
  5. Lancer les migrations et créer l'admin
    python manage.py migrate
    python manage.py createsuperuser
  6. Démarrer le serveur
    python manage.py runserver
    # Ou double-cliquez sur run_local.ps1
    Accédez à http://127.0.0.1:8000

11 Déploiement Docker (production)

Le fichier docker-compose.yml démarre l'ensemble de la stack en une commande.

11.1 Services inclus

ServiceRôlePort
webApplication Django (Gunicorn)8000
qclusterWorker tâches planifiées (django-q2)—
dbBase de données PostgreSQL 165432
zabbix-serverServeur de supervision Zabbix 6.410051
zabbix-webInterface web Zabbix8080
uptime-kumaMonitoring services web3001

11.2 Démarrer la stack

# Copier et remplir le fichier d'environnement
cp .env.example .env

# Démarrer tous les services
docker compose up -d

# Voir les logs
docker compose logs -f web
docker compose logs -f qcluster

# Arrêter
docker compose down

Au premier démarrage, les migrations sont appliquées automatiquement et les tâches planifiées sont enregistrées. Aucune action manuelle n'est nécessaire.

11.3 Créer le premier administrateur

Deux méthodes :

# Méthode 1 : Variable d'environnement (automatique au démarrage)
DJANGO_BOOTSTRAP_ADMIN_PASSWORD=motdepasse_fort
DJANGO_BOOTSTRAP_ADMIN_EMAIL=admin@votrefai.com

# Méthode 2 : Commande manuelle
docker compose exec web python manage.py createsuperuser

12 Tâches automatiques (planificateur)

Le service qcluster exécute trois tâches récurrentes :

TâcheFréquenceCommande manuelle
Rapports revendeurs quotidiens Chaque nuit à 01h00 python manage.py generate_daily_revendeur_reports
Synchronisation santé PtP (Zabbix) Toutes les 10 minutes python manage.py sync_ptp_health --dry-run
Synchronisation hotspot MikroTik Toutes les 30 minutes python manage.py sync_mikrotik_hotspot

Surveiller les tâches

Depuis l'interface Admin Django : /admin/django_q/

Vous pouvez y voir l'historique d'exécution, les erreurs et modifier les plannings.

Options utiles des commandes

# Rapport pour une date spécifique
python manage.py generate_daily_revendeur_reports --date 2026-06-15

# Limiter à un revendeur (organisation)
python manage.py generate_daily_revendeur_reports --tenant mon-fai

# Tester la sync PtP sans modifier la base
python manage.py sync_ptp_health --dry-run

# Voir les tâches planifiées
python manage.py setup_schedules --list
